A rule has four parts

1 Source

Where the event comes from, like Windows, cloud audit, identity, or network logs.

2 Conditions

The field checks that describe suspicious behavior, such as command line contains a risky keyword.

3 Logic

Whether every condition must match or just one condition is enough to alert.

4 Tuning

How you reduce noisy alerts with exclusions, severity, and a clear investigation path.
